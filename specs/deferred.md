# Deferred Work

Work that has been identified and triaged but explicitly set aside — not ignored, not blocked, just not now. Distinct from:

- `findings.md` — the audit trail of what was *observed* (append-only, never actioned here)
- `escalations.md` — work that is *blocked* pending a user decision
- `fidelity-index.md` — coverage gaps that will surface as tasks when a sweep runs

Items here are **active choices** to defer. Each entry should explain why it was deferred and what condition would move it to active work.

---

## Format

```
### DEF-NNN: <Short title>

**Source:** findings.md entry | sweep | user | design session | ADR-NNNN
**Severity / urgency:** CRITICAL | HIGH | MEDIUM | LOW
**Deferred because:** <why not now>
**Activate when:** <condition that should trigger picking this up>
**Added:** YYYY-MM-DD
```
