# Architecture Decision Records

One file per decision. Filename: `NNNN-<kebab-case-title>.md` (e.g. `0002-use-event-sourcing.md`).

## Status Values

| Status | Meaning |
|--------|---------|
| `Proposed` | Draft — under adversary review, not yet finalized |
| `Accepted` | Finalized — governs current implementation |
| `Superseded by NNNN` | Replaced by a later decision; kept for history |
| `Rejected` | Considered and explicitly not adopted; kept so it is not re-litigated |

## When to Write an ADR

- Any non-trivial design decision that affects the domain model, a public interface, or an invariant
- Any decision that took meaningful deliberation — so future readers understand why, not just what
- Any decision that was controversial, where the rejected options matter as context
