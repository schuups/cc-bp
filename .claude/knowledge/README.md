# Knowledge Folder

This folder is Claude's working memory for the project. Two subfolders, two loose files.

---

## Folders

**`context/`** — what the project is and must be; initialised by `/onboard`, updated as the project evolves.
- `project-context.md` — purpose, users, constraints, execution context
- `requirements.md` — functional requirements inventory (implemented / planned / deferred)
- `invariants.md` — properties that must always hold; derived from requirements and ADRs
- `architecture.md` — system boundaries, components, key decisions
- `adr/` — architecture decision records explaining why architecture.md looks the way it does

**`health/`** — how the project is doing; updated continuously during development.
- `fidelity-index.md` — coverage ratings per feature area (THOROUGH / MODERATE / LOW / NONE)
- `adversary-findings.md` — issues found during adversary reviews of designs and implementations
- `audit-log.md` — auditor sign-offs per changeset
- `escalations.md` — blocked items awaiting human resolution
- `integrity-log.md` — results of `/integrity` structural audits

**`coding-guide.md`** — language-specific coding rules generated by `/onboard`; not context about what the project is, but how to develop it.

**`deferred.md`** — backlog of work explicitly set aside; read by Claude at session start to avoid re-proposing what was already deferred.

---

## Flow — new functional requirement

```
context/requirements.md     add requirement (status: planned)
  → context/adr/            new ADR: design options, adversary review, decision
  → context/architecture.md update if system shape changes
  → context/invariants.md   add any new invariants the design introduces
  → [code written]
  → health/fidelity-index.md  new entry, updated to THOROUGH after implementation
  → health/adversary-findings.md  issues logged during implementation review
  → health/audit-log.md     auditor sign-off recorded
  → context/requirements.md update status: implemented
  → /docs                   rebuild documentation
```

## Flow — bug

```
health/adversary-findings.md or health/escalations.md  record the bug
  → [failing test written, then fix]
  → health/adversary-findings.md  review of fix
  → health/audit-log.md     sign-off
```
