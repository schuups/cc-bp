# Claude Code Blueprint

Defines the working agreements, roles, modes, protocols, rules, and guardrails for all Claude Code interactions on software projects.

---

## Knowledge Folder — `.claude/knowledge/`
See `knowledge/README.md` for flow and lifecycle.

`context/` — what the project is and must be:
- `project-context.md` — purpose, users, constraints, execution context
- `requirements.md` — functional requirements inventory (implemented / planned / deferred)
- `invariants.md` — properties that must always hold; derived from requirements and ADRs
- `architecture.md` — system boundaries and key decisions
- `adr/` — architecture decision records; `INDEX.md` lists all ADRs; `0001-template.md` is the template for new ones

`health/` — how the project is doing:
- `fidelity-index.md` — coverage ratings per feature area (THOROUGH / MODERATE / LOW / NONE)
- `adversary-findings.md` — issues found during adversary reviews
- `audit-log.md` — auditor sign-offs per changeset
- `escalations.md` — blocked items awaiting human resolution
- `integrity-log.md` — results of `/integrity` structural audits

`coding-guide.md` — language-specific rules (generated by `/onboard`)

`deferred.md` — deferred work backlog; read by Claude at session start

---

## Commands

| Command | Description | Definition |
|---------|-------------|------------|
| `/onboard` | Personalises this blueprint for the specific project; self-removes after completion | `commands/onboard.md` |
| `/status` | Dashboard snapshot of project health; read-only; runs automatically at session start | `commands/status.md` |
| `/integrity` | Active structural audit: broken links, stale docs, unregistered work items, spec inconsistencies; run on demand | `commands/integrity.md` |
| `/docs` | Rebuilds root `documentation/` and root `README.md` from `.claude/knowledge/` | `commands/docs.md` |
